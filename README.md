# 🚀 Ultimate Media Suite v1.3.2

Ultimate Media Suite เป็นแอปพลิเคชันเดสก์ท็อปประสิทธิภาพสูงสำหรับการดาวน์โหลด ดึงข้อมูล และแปลงไฟล์วิดีโอ เสียง และภาพเคลื่อนไหว พัฒนาขึ้นด้วยอินเทอร์เฟซผู้ใช้แบบเดสก์ท็อปที่สวยงาม ทันสมัย รองรับ Dark Mode โดยทำงานร่วมกับเครื่องมือระดับอุตสาหกรรมอย่าง `yt-dlp` และ `FFmpeg`

ตัวโปรแกรมถูกออกแบบขึ้นอย่างประณีต มีระบบคิวสำหรับการดาวน์โหลดแบบขนาน (Multi-threaded), การดึงเซสชันคุกกี้จากเบราว์เซอร์, การเร่งความเร็วด้วยฮาร์ดแวร์ GPU ครบครันทุกค่ายหลัก (NVIDIA, Intel, AMD) และมีสถาปัตยกรรมการแจกจ่ายโปรแกรมแบบพกพาที่ไม่จำเป็นต้องติดตั้ง (Portable Model)

## 🛠️ เทคโนโลยีที่ใช้และไลบรารีที่เกี่ยวข้อง (Tech Stack & Dependencies)

* **ภาษาหลัก**: Python 3.14+
* **ส่วนประสานงานผู้ใช้ (GUI Framework)**: CustomTkinter (ไลบรารีที่พัฒนาต่อจาก Tkinter ให้มีสไตล์ HSL ที่สวยงามและรองรับธีมมืดอย่างสมบูรณ์)
* **เครื่องยนต์หลัก (Engines)**: 
  * `yt-dlp` (เครื่องมือสแกนและดาวน์โหลดกระแสข้อมูลวิดีโอออนไลน์ประสิทธิภาพสูง)
  * `FFmpeg` / `FFprobe` (เครื่องมือมาตรฐานระดับอุตสาหกรรมสำหรับการประมวลผลมัลติมีเดียและวิเคราะห์ข้อมูลเมตาดาต้า)
* **ไลบรารีและยูทิลิตี้**:
  * `concurrent.futures` / `threading` (การบริหารจัดการคิวงานแบบอะซิงโครนัสและการรันเธรดดาวน์โหลดขนาน)
  * `Pillow` (การประมวลผลภาพและย่อภาพหน้าปก Thumbnail สำหรับการแสดงพรีวิวในแอป)
  * `subprocess` (การเชื่อมโยงระบบการทำงานระหว่างโปรเซสหลักและโปรแกรมภายนอกในคิวคำสั่ง)
  * `uv` (ตัวจัดการแพ็คเกจไพทอนเจเนอเรชันใหม่ เพื่อการสร้างและจัดเตรียมสภาพแวดล้อมที่รวดเร็วและสะอาดที่สุด)
* **การแพ็คเกจโปรแกรม (Packaging)**: `PyInstaller` (บิลด์ออกมารันเป็นไฟล์เดี่ยวแบบ Portable Executable โดยไม่ต้องติดตั้งตัวโปรแกรมลงในระบบ)

## 🌟 คุณสมบัติเด่นของโปรแกรม (Key Features)

### 📥 ระบบดาวน์โหลดสตรีมความเร็วสูง (High-Speed Stream Downloader)
* **การรองรับที่กว้างขวาง**: สนับสนุน YouTube, Facebook, Instagram, TikTok และแพลตฟอร์มยอดนิยมอื่นๆ ที่ขยายขอบเขตโดยแกนของ yt-dlp
* **ความละเอียดสูงสุด**: ดาวน์โหลดคลิปได้ละเอียดสูงถึงระดับ **4K (2160p)** พร้อมระบบผสานวิดีโอและออดิโอให้อัตโนมัติอย่างแนบเนียน
* **การดึงเสียงคุณภาพสูง**: แยกข้อมูลเสียงออกมาเป็นไฟล์คุณภาพเสียงระดับสตูดิโอ เช่น **MP3 (Ultra 320kbps)** หรือ **WAV** พร้อมฝังข้อมูล Tags ให้ครบถ้วน
* **การจัดระเบียบไฟล์อัจฉริยะ**: แยกไฟล์ตามชื่อเจ้าของช่อง (Channel Name) สร้างโฟลเดอร์แยกหมวดหมู่ย่อย (`Videos/Thumbnails`) และตั้งชื่อไฟล์ด้วยลำดับรันเบอร์อย่างเรียบร้อย
* **การวิเคราะห์แบบไม่หน่วงหน้าจอ**: สแกนหาข้อมูลวิดีโอแบบเบื้องหลัง (Asynchronous Stream Checking) ช่วยให้แอปไม่ค้างชั่วขณะระหว่างประมวลผลข้อมูลลิงก์
* **โหมดดาวน์โหลดเพลย์ลิสต์/โปรไฟล์**: รองรับการดาวน์โหลดเพลย์ลิสต์ขนาดใหญ่ หรือหน้าโปรไฟล์วิดีโอทั้งหมดพร้อมกันแบบ Batch
* **การเชื่อมต่อคุกกี้เบราว์เซอร์**: ดึงคุกกี้มาช่วยสแกนจากเบราว์เซอร์หลักในเครื่อง (Chrome, Edge, Firefox, Brave, Opera, Vivaldi, Safari) เพื่อดาวน์โหลดวิดีโอที่เป็นส่วนตัวหรือติดจำกัดอายุ

### 🔄 ระบบแปลงไฟล์เร่งความเร็วด้วย GPU ครบทุกค่าย (Multi-Vendor Hardware-Accelerated Converter)
* **ฟอร์แมตปลายทาง**: รองรับ MP4, MKV, MP3, WAV, GIF, FLAC, AAC, WEBM
* **ระบบตรวจจับการ์ดจออัตโนมัติ**: ตรวจหาคุณสมบัติการเร่งความเร็วของการ์ดจอในคอมพิวเตอร์ครั้งเดียวตอนเปิดโปรแกรมและเก็บเป็นแคช สนับสนุน GPU ค่ายหลักทั้งหมด:
  * **NVIDIA**: เข้ารหัสผ่านเทคโนโลยี `h264_nvenc`
  * **Intel Quick Sync**: เข้ารหัสผ่านเทคโนโลยี `h264_qsv`
  * **AMD**: เข้ารหัสผ่านเทคโนโลยี `h264_amf`
* **ตัวสำรองระบบเสถียร (CPU Fallback)**: ในกรณีที่ระบบไม่มีการ์ดจอที่รองรับ จะปรับมาเข้ารหัสด้วย CPU อย่างปลอดภัยผ่านตัวเข้ารหัสมาตรฐานเสถียรสูงระดับสากล (`libx264` + `aac`) เพื่อรับประกันว่าวิดีโอที่ได้จะลื่นไหลและเปิดได้ทุกอุปกรณ์
* **ความละเอียดและคุณภาพ**: มีโปรไฟล์คุณภาพให้เลือกแปลงครบถ้วน ทั้งระดับ Ultra (CRF 18 / slow preset), Normal (CRF 23 / medium preset) และ Small (CRF 28 / ultrafast preset)

### ⚙️ การตั้งค่าและปรับจูนความต้องการ (Settings & Configuration)
* **การดาวน์โหลดขนานปรับระดับได้**: ผู้ใช้สามารถระบุจำนวนงานที่ดาวน์โหลดพร้อมกันได้ตามใจชอบตั้งแต่ **1 ถึง 10 เธรดขนาน** เพื่อจัดสรรความแรงเครือข่ายให้เหมาะสมที่สุด
* **กำหนดโฟลเดอร์ผลลัพธ์**: ตั้งค่าและบันทึกพาธเริ่มต้นในการเก็บไฟล์ผลลัพธ์ได้อย่างอิสระ
* **อัปเดตแกนโปรแกรมอัจฉริยะ**: ปุ่มอัปเดตด้วยคลิกเดียว เพื่อดาวน์โหลดเวอร์ชันล่าสุดของตัวแกนดึงข้อมูล `yt-dlp` ทันที ป้องกันปัญหาลิงก์เสียเมื่อแพลตฟอร์มออนไลน์เปลี่ยนโครงสร้างวิดีโอ

## 💡 เบื้องหลังการออกแบบและบทเรียนที่ได้รับ (Problem Solving & Design Decisions)

### 1. การแก้ไขปัญหา Network Bottlenecks และจำกัดการเข้าถึง (HTTP 403 Bypass)
* **ความท้าทาย**: ในช่วงเริ่มต้น เมื่อดาวน์โหลดวิดีโอจากเพลย์ลิสต์ขนาดใหญ่ที่มีจำนวนไฟล์ 50 คลิปขึ้นไป โปรแกรมจะสร้างเธรดใหม่เพื่อดึงภาพพรีวิวหน้าปก Thumbnail พร้อมๆ กันทันที ซึ่งการสแปมคิวพร้อมกันแบบไร้ทิศทางส่งผลให้พอร์ตเครือข่ายเต็ม แอปค้างชั่วขณะ และเกิดข้อผิดพลาด 403 Forbidden จากการที่ระบบปลายทาง (เช่น YouTube) บล็อกเนื่องจากขอข้อมูลถี่เกินไป
* **แนวทางแก้ไข**: นำสถาปัตยกรรมแบบควบคุมคิว `ThreadPoolExecutor(max_workers=4)` เข้ามาจัดการควบคุมขั้นตอนการดาวน์โหลดรูปภาพ โดยจำกัดการประมวลผลพร้อมกันไว้ครั้งละไม่เกิน 4 รูปขนานกัน
* **บทเรียนที่ได้รับ**: ได้เรียนรู้ถึงความสำคัญของการจัดการทรัพยากรเครือข่าย เธรดพูล และการออกแบบตัวป้องกันการบล็อกจากระบบเครือข่ายภายนอก (Rate-Limit Boundaries) ในสายงานอัตโนมัติ

### 2. การลดภาระระบบด้วยการเลี่ยงตรวจสอบโปรเซสซ้ำซ้อน (Subprocess Polling Elimination)
* **ความท้าทาย**: แต่เดิม แอปพลิเคชันทำการตรวจสอบความสามารถของการ์ดจอในการรองรับฮาร์ดแวร์แอคเซอเลอเรเตอร์โดยการรันโปรเซสย่อย `ffmpeg -encoders` ทุกครั้งที่กดเริ่มแปลงไฟล์ ซึ่งสิ้นเปลืองเวลา CPU และหน่วงการทำงานก่อนแปลงไฟล์ไปถึง 300ms–800ms ต่อไฟล์
* **แนวทางแก้ไข**: ปรับปรุงให้ตัวแอปพลิเคชันทำการเรียกเช็คข้อมูลผ่านโปรเซสในเบื้องหลังเพียงครั้งเดียวตอนระบบเริ่มต้นเปิดตัว แล้วนำผลลัพธ์มาเก็บเป็นแคชข้อมูลสถานะ `self.supported_encoders`
* **บทเรียนที่ได้รับ**: การจัดเก็บแคชข้อมูลระบบและสแกนล่วงหน้ามีผลอย่างมากต่อประสิทธิภาพการตอบสนองของแอปพลิเคชัน (Latency Optimization) ช่วยให้ผู้ใช้รู้สึกว่าโปรแกรมลื่นไหลขึ้น

### 3. การประสานสเปกการ์ดจอข้ามแพลตฟอร์มอย่างเต็มพิกัด (Cross-Vendor GPU Optimization)
* **ความท้าทาย**: ผู้ใช้ที่รันโปรแกรมบนคอมพิวเตอร์ที่ไม่มีการ์ดจอค่าย NVIDIA จะไม่สามารถใช้ความสามารถการเร่งความเร็วประมวลผลการแปลงไฟล์ได้ เนื่องจากระบบเดิมเขียนล็อกสเปกไว้เฉพาะ NVENC
* **แนวทางแก้ไข**: เขียนการตรวจจับ GPU ให้ยืดหยุ่น โดยดึงรหัสของ **Intel Quick Sync (`h264_qsv`)** และ **AMD AMF (`h264_amf`)** มาแปลงพารามิเตอร์ และประยุกต์ใช้ในการตั้งค่าคุณภาพและการคุมอัตราบิตเรต (`global_quality` vs. `qp` vs. `cq`) ที่เจาะจงให้สัมพันธ์กับสถาปัตยกรรมสเปกฮาร์ดแวร์การ์ดแต่ละค่ายอย่างแท้จริง
* **บทเรียนที่ได้รับ**: ได้พัฒนาความเข้าใจเกี่ยวกับการออกแบบระบบให้มีความยืดหยุ่นต่อสิ่งแวดล้อมฮาร์ดแวร์ที่แตกต่างกัน รวมถึงทักษะด้านการแปลงพารามิเตอร์ของระบบมัลติมีเดียขั้นสูง

### 4. สถาปัตยกรรมการแจกจ่ายแอปแบบไฟล์เดี่ยวและการจัดเส้นทางไฟล์ชั่วคราว (`_MEIPASS`)
* **ความท้าทาย**: เมื่อรวมไฟล์ด้วย PyInstaller เข้าเป็นไฟล์เดี่ยวแบบ Portable ตัวระบบจะไม่สามารถทำงานและเรียกใช้เครื่องยนต์ดาวน์โหลดเสริม (`ffmpeg.exe`, `yt-dlp.exe`) ได้ เนื่องจากเส้นทางไฟล์ปกติจะชี้ไปยังตำแหน่งไฟล์ปัจจุบัน แทนที่จะเป็นโฟลเดอร์ชั่วคราวในหน่วยความจำที่ PyInstaller แตกไฟล์ออกมา
* **แนวทางแก้ไข**: ออกแบบฟังก์ชัน `resource_path()` ให้ค้นหาและชี้พารามิเตอร์ไปยังไดเรกทอรีชั่วคราว `sys._MEIPASS` ของ PyInstaller เมื่อโปรแกรมทำงานในลักษณะคอมไพล์สำเร็จแล้ว เพื่อให้เครื่องยนต์เสริมทำงานได้อย่างราบรื่น
* **บทเรียนที่ได้รับ**: ความเข้าใจลึกซึ้งเกี่ยวกับการบิลด์โปรแกรมเดสก์ท็อป การทำงานในสภาวะกล่องทรายชั่วคราว (Sandbox Environments) และการบริหารจัดแจง Absolute Path ในไฟล์ไบนารีที่แจกจ่าย

## ⚙️ ขั้นตอนการติดตั้งและรันใช้งาน (Installation & How to Run)

### เครื่องยนต์ไบนารีเบื้องต้น (Prerequisite Engines)
โปรดตรวจสอบว่ามีไฟล์ `ffmpeg.exe`, `ffprobe.exe` และ `yt-dlp.exe` อยู่ในโฟลเดอร์ `bin/` หรือเพิ่มโฟลเดอร์เหล่านั้นเข้าสู่ระบบ `PATH` ของคอมพิวเตอร์ท่านแล้ว

### การทดสอบรันจากซอร์สโค้ด (Development Setup)
การรันผ่านเครื่องมือจัดเตรียมยุคใหม่อย่าง **uv** (แนะนำ - ติดตั้งรวดเร็วและสะอาดที่สุด):
```bash
# คัดลอกโฟลเดอร์และเปิดเข้าไปในไดเรกทอรี
cd -Ultimate-Media-Suite-main

# สร้างสภาพแวดล้อมจำลองและติดตั้งความต้องการอัตโนมัติ
uv venv
uv pip install -r requirements.txt

# รันเริ่มโปรแกรมได้ทันที
uv run python main.py
```

การรันผ่านเครื่องมือ Python มาตรฐาน:
```bash
# สร้างสภาพแวดล้อมจำลอง (Virtual Environment)
python -m venv .venv
.venv\Scripts\activate

# ติดตั้งไลบรารีที่จำเป็น
pip install -r requirements.txt

# รันแอปพลิเคชัน
python main.py
```

### การบิลด์แพ็คเกจตัวรันแบบเดี่ยว (Packaging to Standalone Executable)
รันสคริปต์อัตโนมัติเพื่อล้างโฟลเดอร์เก่าและดำเนินการสร้างไฟล์ตัวรัน `.exe` ได้ทันที:
```bash
.\build.bat
```
เมื่อเสร็จสมบูรณ์ ตัวโปรแกรมจะถูกจัดเตรียมพร้อมส่งมอบในไดเรกทอรี `dist/Ultimate Media Suite.exe`
